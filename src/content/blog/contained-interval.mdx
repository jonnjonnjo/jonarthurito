## Background

Given $N$ $(1 \leq N \leq 10^5)$ pair of interval $L$ and $R$ $(1 \leq L \leq R \leq 10^9)$, we want to filter it
such that the remaining Intervalerval is not pair-containing. Interval $P_i$ and $P_j$ could be called $\mathbf{pair-containing}$ if and only if 
$P_i.L \leq P_j.L \leq P_j.R \leq P_i.L$. We want to filter it so that there exists no pair $(i,j)$ such that $P_j$ and $ P_i$ are $\mathbf{pair-containng}$ .


## Idea 
### Precompute 

Let's call the list of the interval $V$. Let's first sort $V$ such that : 
1. The $L$ point will be non-decreasing 
2. The $R$ point will be non-increasing 


That will resulted in $V$ that has an non-decreasing left-point and non-increasing right-point if the left-point happened to coincided. 
For example, let's try it on $V=[(1,5),(6,12),(6,15),(7,8),(14,20)]$ . After sorting it, we will get $V=[(1,5),(6,15),(6,12),(7,8),(14,20)]$

### Filtering

The general idea is that as we iterate through the V, we want to know whether there exists a previously visited interval such that such interval are $\mathbf{pair-containg}$ with the 
currently iterated interval.
Let's iterate through all element in the sorted $V$ and denotes the current element as $x$. As we iterate throuh each $x$, we want to know whether there exists another $P$ 
such that the interval $x$ belongs inside $P$. We know that we have visited all intervals $P$ of which $P.L \leq x.L$.. Because we already know that all such intervals $P$ satisfy $P.L \leq x.L$, then we only want to know 
the value of highest $P.R$ across all visited interval.

let's call a randomly choosen previous interval $u$. Because we already know that $u.L \leq x.L$, then we want to maximize the $u.R$ so that it might exceeds 
the current rightmost segment of current element or might exceeds the $x.R$. More formally, we want to maximixe $u.R$ such that we could prove that we could 
satisfy $x.R \leq u.R$ which fulfills the requirement for $x$ to be contained in another interval.


## Pseudocode

Here is a simple Pseudocode for such algorithms. As it can be seen, the time complexity for the algorith is $O(N)$ because we are only doing 
one iteration and trivial comparison for each of it. Unless the $V$ hasn't been sorted yet, the time complexity will be $O(N \log (N))$
<Pseudocode>
```latex
\begin{algorithm}
\caption{Filtering pair-containing intervals}
\begin{algorithmic}
\REQUIRE Sorted interval list $V$
\ENSURE No pair-containing intervals remain
\STATE $maxR \gets -\infty$
\FOR{each $x \in V$}
    \IF{$x.R \le maxR$}
        \STATE remove $x$    \COMMENT{Interval is contained}
    \ELSE
        \STATE $maxR \gets x.R$
    \ENDIF
\ENDFOR
\end{algorithmic}
\end{algorithm}
```
</Pseudocode>


## Caveat 

It should be noted that, after the $V$ filtered, the $|V|$ will be minimzed because we filter out those child intervals. 
We can try to maximize $|V|$ by trying to remove the parent interval instead of the child one.
